OPIS OGOLNY
============
Bot do automatycznego lowienia ryb w grze Metin2 (serwer Eryndos).
Wersja POST CNN - oparta na lekkiej sieci neuronowej FishNet (CNN)
z fallbackiem na klasyczną detekcje (background subtraction).

CNN dziala na POJEDYNCZEJ klatce — bez historii, bez bufora, bez warmup.
Eliminuje glowne problemy wersji PRE CNN:
  - Rozpoznaje napisy HIT/MISS jako osobne klasy
  - Nie degraduje skutecznosci w czasie
  - Natychmiastowa detekcja od 1. klatki

ARCHITEKTURA CNN (FishNet)
===========================
  - Custom lightweight CNN: 4 bloki conv + GAP + 2 heads
  - Parametry: ~113K (~0.5 MB float32, ~115 KB int8)
  - Input: 128x128x3 (RGB, normalizowane 0-1)
  - Output 1: Klasyfikacja stanu (5 klas)
    * INACTIVE  = brak minigry
    * WHITE     = bialy okrag (czekaj)
    * RED       = czerwony okrag (klikaj!)
    * HIT_TEXT  = napis HIT (ignoruj)
    * MISS_TEXT = napis MISS (ignoruj)
  - Output 2: Regresja pozycji rybki (x, y, confidence)
  - Inferencja: ONNX Runtime, <5ms na CPU (1 watek)

PIPELINE TRENINGOWY
====================
  1. Zbierz klatki: test10_clean/raw/ (~268), nowe nagranie (~2000+)
  2. Pseudo-labels: python -m cnn.generate_pseudo_labels
  3. Reczne etykietowanie: python -m cnn.label_tool --frames <folder>
  4. Trening: python -m cnn.train (AdamW, OneCycleLR, 80 epok)
  5. Eksport: python -m cnn.export_onnx
  6. Test: python run.py --version post_cnn --debug

URUCHOMIENIE
=============
  Z CNN (domyslne):
    python run.py --version post_cnn --debug

  Bez CNN (fallback klasyczny):
    python run.py --version post_cnn --debug --no-cnn

  Wymaga:
    pip install onnxruntime  (do inferencji CNN)
    Model ONNX w cnn/models/fishnet_int8.onnx (lub fishnet.onnx)

OCZEKIWANE USPRAWNIENIA (vs PRE CNN)
=====================================
  MISS_TXT:    20% bledow -> ~2% (osobna klasa CNN)
  BRAK:        26.7% -> ~5% (single-frame, bez warmup)
  POZA:        13.3% -> ~8% (regresja centroida)
  Degradacja:  62%->34% -> stala ~80%+ (stateless)
  Warmup:      3+ klatki -> 0 klatek (natychmiastowa)
  Czas det.:   5-15ms -> 3-5ms (ONNX Runtime)
  Pamiec:      15 klatek bufor -> 0 (zero state)
  Kod det.:    639 linii -> ~50 linii inferencji
